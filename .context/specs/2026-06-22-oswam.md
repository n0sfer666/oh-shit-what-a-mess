# Спецификация: OSWaM (oh-sht-what-a-mess)

> Консольная (TUI) утилита для анализа и **безопасной** очистки занятого места на macOS.
> Дата: 2026-06-22. Статус: draft. Источник: spec-interview.

## Контекст

Пользователю нужен лёгкий opensource-инструмент, который находит «мусор», который
коллекционирует macOS (кэши, dev-артефакты, Docker, логи, корзина), показывает где
занято место и помогает безопасно его освободить.

**Ключевой инвариант: БЕЗОПАСНОСТЬ.** Утилита никогда не должна кирпичить систему или
удалять то, что удалять нельзя. Любое деструктивное действие — только по явному
подтверждению пользователя. Вторая по важности цель — UX (красивый, понятный TUI).

### Данные реальной машины (контекст инвестигации, на момент интервью)
- macOS 26.5.1, Apple Silicon, SIP **включён**. Диск: ~303 GB занято / ~140 GB свободно.
- `~/Library/Caches` ≈ **30 GB** (Yarn 8.5G, Homebrew 4.3G, Google/Chrome 4G, Playwright 2.3G, Arc 1.9G, Spotify 1.8G, …).
- `Docker.raw` ≈ **1.0 TB логически / ~25 GB физически** (sparse). `CoreSimulator` 8G, `~/.npm` 3.7G, `~/.cache` 3.4G.

## Требования

### Функциональные
1. **Сканирование** известных локаций мусора (гибрид: быстрый таргетный скан по реестру
   путей по умолчанию + опциональный полный обход ФС для поиска крупных неизвестных файлов).
2. **Классификация риска** каждого элемента по 4 уровням (Safe / Caution / Danger / Never)
   на основе набора сигналов (см. «Движок риска»).
3. **TUI** для просмотра по категориям, описания риска, drill-down в список файлов,
   выбора (категория целиком и/или отдельные файлы) и запуска удаления.
4. **CLI** (неинтерактивный режим): `--json`, `--dry-run`, выбор категорий флагами — для скриптов.
5. **Удаление** выбранного: в момент нажатия «Удалить» спросить **в Корзину или безвозвратно**.
6. **Docker**: делегировать `docker` CLI (`docker system df` для оценки, `docker system prune`
   для очистки). **Никогда** не трогать `Docker.raw` напрямую.
7. **Dev-кэши**: гибрид — родные команды там, где безопаснее (docker; опц. brew),
   прямое удаление файлов для остального.
8. **Защита от живых процессов**: детектить, держит ли запущенный процесс путь
   (running apps / lsof), помечать «Осторожно: приложение запущено».
9. **Расчёт размера**: **физический (on-disk)** размер (реально освобождаемое место),
   не логический (иначе sparse/clones обманывают).
10. **Защитная сеть**: `--dry-run` (превью) + лог-манифест удалённого (что/куда/когда).
11. **Кастомизация защиты**: неизменяемый встроенный `never`-список + пользовательский config
    (protected-paths, ignore-globs).
12. **Help-оверлей** по `?`; показывается при первом запуске.
13. **Safe-by-default**: предвыбраны только Safe-элементы; Caution/Danger требуют явного действия.

### Нефункциональные
- **Язык**: Rust (один статический бинарь, без runtime-зависимостей у пользователя).
- **Лёгкость**: не грузить железо; быстрый таргетный скан по умолчанию.
- **Платформы**: Apple Silicon + Intel (universal binary), последние 2-3 major macOS.
- **Лицензия**: MIT. Opensource, дружелюбно к контрибьюторам.
- **Методология**: TDD (red → green → refactor). `cargo test` зелёный = условие коммита.
- **Модульность**: Cargo workspace, ядро без UI.

## Архитектура

Cargo workspace:

```
oswam/
├── oswam-core/   # чистая логика: scan, classify (risk engine), size (physical),
│                 # process-detection, delete (за trait FsOps), manifest, rules/config.
│                 # НОЛЬ зависимостей от UI. Максимум юнит-тестов.
├── oswam-tui/    # ratatui: layout, темы, навигация, help-оверлей, рендер.
│                 # Тесты через ratatui TestBackend + insta (snapshot).
├── oswam-cli/    # точка входа: парсинг флагов, --json/--dry-run, запуск TUI или batch.
└── ...
```

- **FsOps trait** — абстракция файловых операций (stat/list/move-to-trash/remove).
  Реальная реализация + mock/tmpdir-реализация для тестов. Деструктив тестируется
  ТОЛЬКО на tmpdir-фикстурах, никогда на реальных путях.
- **Реестр категорий** — декларативные правила (путь, тип очистки, дефолтный риск,
  native-команда если есть). Расширяемо.

### Движок риска (ядро безопасности)
4 уровня: **Safe / Caution / Danger / Never**. Уровень вычисляется из сигналов:
- путь относительно реестра / встроенного `never`-списка;
- SIP-защита, владелец `root`;
- dataless / iCloud-стаб;
- держит ли путь запущенный процесс;
- пользовательские protected-paths / ignore-globs.

`Never` (жёстко, не удаляется и предвыбор невозможен): `/System` и SIP · root-owned ·
`sleepimage`/swapfile · внутренности `*.photoslibrary`, Mail envelope index · iCloud Drive
и dataless-стабы · `Containers`/`Group Containers` (данные sandboxed-приложений).

### Категории v1
- **Системный мусор**: user/app caches (`~/Library/Caches/*` — содержимое, не папки),
  логи, корзина, temp, QuickLook/font cache.
- **Dev-окружение**: npm/yarn/pnpm/brew/go/cargo/pip кэши + Xcode DerivedData/CoreSimulator
  + Docker (через CLI).
- **Браузеры**: кэши Chrome/Arc/Comet/Safari/Yandex (только cache, НЕ профили/пароли).
- **Большие данные (только инфо)**: iOS-бэкапы (MobileSync), TM local snapshots, крупные
  файлы — показывать размер, помечать Caution/Never, не предвыбирать к удалению.

## API / Интерфейсы (CLI)
- `oswam` — запуск TUI.
- `oswam --dry-run` — превью без удаления.
- `oswam --json` — машинный вывод результата скана (для скриптов).
- Флаги выбора категорий для batch-режима (детализировать при реализации).

## UI/UX

Референс макетов: `docs/mockups/` (тёмная и светлая темы). Layout:
- **Левый сайдбар**: карточки категорий — компактный глиф-иконка + имя + физ. размер +
  чекбокс + цвет риска и **символ** (✓ Safe / ▲ Caution / ⛔ Never). Цвет + символ всегда
  вместе (дальтонизм). **Группировка переключаемая хоткеем**: по категории / по размеру / по риску.
- **Правый верх — Description**: уровень риска + ПОЧЕМУ безопасно/опасно + какая именно
  команда/действие выполнится (напр. «будет вызван `docker system prune`», а не удаление `.raw`).
- **Правый низ — Table of files**: `name | capacity (physical)`; выбор отдельных файлов.
- **Низ слева — action**: «Ctrl+P to proceed» (запуск удаления → модалка Корзина/Безвозвратно
  + итог «освободится X физически»).
- **Фокус-рамка**: подсветка активной панели.

### Темы
Авто-детект фона терминала (best-effort: OSC 11 / `COLORFGBG`) + переопределение в config +
быстрый toggle хоткеем. Тёмная/светлая.

### Навигация (vim + стандарт)
- Движение: `j/k` + `↑/↓`, `h/l`, `g/G`, `Ctrl+d/Ctrl+u`.
- `Tab` — переключение панелей. `Space` — чекбокс. `Ctrl+P` — proceed (удаление).
- `?` — help-оверлей. `q` — выход.

### Help-оверлей
Модалка поверх UI. **Показывается при первом запуске.** Закрывается любой клавишей,
**кроме** `↑↓` и `hjkl` (они скроллят содержимое справки).

### ASCII-арт
Компактный глиф на категорию (1-2 строки) — больше категорий на экране, лучше скан.

## Edge Cases
- Sparse-файлы / APFS-клоны → физический размер ≠ логический (Docker.raw 1TB/25GB).
- Запущенное приложение держит кэш → пометка Caution, риск коррапта.
- iCloud dataless-стаб → Never (удаление = потеря/перекачка).
- Удаление в Корзину НЕ освобождает место сразу (до очистки корзины) — честно сообщать.
- Symlink/TOCTOU при обходе ФС — не следовать по симлинкам при удалении; проверять перед rm.
- Опциональный sudo для системных кэшей (`/Library/Caches`) — отдельный opt-in флаг + доп. предупреждения.
- Терминал без авто-детекта темы / маленький размер окна / отсутствие docker/brew — graceful degrade.

## Риски и митигации
| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Удаление нужных данных | Средняя | 4-уровневый риск; встроенный `never`; safe-by-default; Корзина по умолчанию; dry-run; манифест |
| Кирпич системы | Низкая | user-space по умолчанию; SIP/root неудаляемы; sudo только opt-in |
| Коррапт кэша запущенного приложения | Средняя | детект процессов → Caution-пометка |
| Обман по размеру (sparse/clones) | Высокая | физический (on-disk) расчёт |
| Потеря Docker volumes | Средняя | только `docker system prune` через CLI, не трогать `.raw` |
| iCloud data loss | Средняя | dataless-стабы → Never |
| Ошибочное удаление через симлинк (TOCTOU) | Низкая | не следовать симлинкам, проверка перед операцией |

## Тестовая стратегия (TDD)
- **Уровень**: T2-T4 (деструктив + критичный путь). Каждая фича начинается с падающего теста.
- **oswam-core**: unit-тесты на классификатор риска, парсер физ./лог. размера, детект процессов,
  построение манифеста, dry-run. Деструктив — только на tmpdir-фикстурах через FsOps.
- **oswam-tui**: `TestBackend` (рендер в буфер) + `insta` snapshot-тесты (layout, темы,
  фокус, help-оверлей, цвета/символы риска).
- **Условие коммита**: `cargo test` + `cargo clippy` + `cargo fmt --check` зелёные.

## Решения по открытым вопросам (2026-06-22, перед реализацией)
- **Стек**: ratatui + crossterm (TUI), clap derive (CLI), serde + toml (config), serde_json
  (manifest, `--json`), thiserror (core) + anyhow (cli). Config: `~/.config/oswam/config.toml`.
- **CLI batch — сабкоманды**: `oswam` (TUI) · `oswam scan [--json]` ·
  `oswam clean [--safe] [--category caches,dev,…] [--dry-run] [--trash|--delete]`.
- **Persist**: только config (protected-paths, ignore-globs, theme). Выбор к удалению и
  UI-префы между сессиями НЕ сохраняются (безопасность + простота).
- **sudo**: поддерживается. Без sudo — user-space; `sudo oswam` добавляет: (1) локальные снимки
  Time Machine (удаляются все, КРОМЕ последнего); (2) системные кэши `/Library/Caches` (риск Danger,
  только содержимое, SIP-защищённые пропускаются, безвозвратно, не предвыбираются). `/System/Library/Caches`
  не трогается. Стартовый экран информирует о режиме и рисках.
- **Тема**: best-effort OSC 11 → fallback `COLORFGBG` → fallback dark; override в config + хоткей.
- **Реестр категорий**: декларативный, стартовый набор v1 ниже (расширяемо).

## Открытые вопросы (backlog)
- Точные пути реестра расширяются по мере тестов на реальной машине.
- Надёжность OSC 11 на конкретных терминалах (graceful degrade уже заложен).
- Opt-in `--system` (sudo) — отдельная итерация.
