# Конвенции

- Без комментариев в коде (кроме явного запроса).
- Файлы ≤ 200 строк; одна ответственность — один файл/модуль.
- Строгая типизация; без `unwrap()` на горячем/критичном пути (best-effort через `unwrap_or`).
- Ошибки: core — `thiserror`/`io::Result`; cli — `anyhow`.
- TDD: тест вперёд, `cargo test` зелёный = условие коммита.
- FsOps за trait; деструктив тестируется только на `FakeFs`/tmpdir, никогда на реальных путях.
- TUI: логика состояния отделена от рендера; snapshot-тесты через `TestBackend` + `insta`.
- Безопасность: never-список неизменяем; safe-by-default; Корзина по умолчанию; dry-run + манифест.
- Коммиты: conventional commits, английский.
